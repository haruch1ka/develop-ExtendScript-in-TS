# 開発ルール

## 作業開始の条件

- 作業は、明確にユーザーが指示を与えるまで開始しないこと。

## 開発の流れ

Claude Code とユーザーがプログラム開発をする際は、通常のプログラム開発と同じ流れをたどること。

1. 要件定義
2. 基本設計
3. 詳細設計
4. コーディング
5. テスト
6. 受け入れテスト

## ドキュメントの作成場所

- 要件定義・基本設計・詳細設計は `docs/plans/${project名}/` に作成する。
- 作成するものの前提を細かく取り決める。

## 計画段階での質問の仕方

- 計画や作りたいものに関する情報はユーザーの頭の中にある。計画の段階で不明な点があるときは、各ファイルを参照せず、ユーザーに質問して計画を詰めること。
- 複数質問がある場合は、順に1つずつ質問すること。
